# 1. GitHub에서 자신의 컴퓨터로 작업할 Repository를 Clone하기
## 1.1. clone
먼저, 우리가 작업을 수행할 repository를 자신의 컴퓨터로 옮겨와야 합니다. 통째로 복붙하는 방법도 있겠지만, 상당히 비효율적이니 다른 방법을 사용해봅시다.
> - Github Desktop이라는 프로그램을 다운받아 설치합니다. 구글에 검색하면 바로 다운로드 링크가 나옵니다.
> - 로그인을 합니다.
> - Github 사이트에서 우리의 repository(snumentalhealth_21_1)을 찾아 클릭.
> - 오른쪽에 초록색 버튼으로 'code'라고 표시되어 있을 겁니다. 클릭.
> - 'clone' 항목에서 'HTTPS'를 누르면 주소가 뜰 텐데 통째로 복사합니다.
> - Github Desktop으로 돌아와서, 'File > Clone Repository' 클릭.
> - Snumental 계정으로 로그인이 되어 있다면, Github.com 탭을 누릅니다. 주소를 붙여넣지 않아도 됩니다. 이때 실수로 **snumentalhealth_20_2를 선택하지 않도록 합니다!**
> - 만약 다른 계정으로 로그인되어 있다면, url 탭을 누릅니다. 주소를 붙여넣습니다.
> - Local Path를 선택합니다. 이때, **repository의 이름과 로컬 폴더의 이름이 같아야 합니다.**
> - 끄읏.

## 1.2. commit, push, pull
그 다음으론, commit과 push에 대해 알아보겠습니다.
> - 파일을 수정하고 나면, Github Desktop이 저절로 수정 사항을 기록해 대기시켜 둘 겁니다.
> - summary와 description을 달고, 'Commit to Main'을 누릅니다. **commit**이란, 사용자가 git의 형태로 코드를 포장한다는 의미입니다. 택배를 포장해서 우체국에 맡기는 것과 비슷합니다.
> - 그리고 상단의 'push'를 누릅니다. **push**란, 사용자가 commit된 git을 다른 git 저장소로 전달하는 것을 의미합니다. 택배를 배송하는 것과 비슷합니다.
> - summary와 description은 항상 꼼꼼하게 적어야 합니다. 이걸 해야 대략 어떤 것을 했는지 잘 알 수 있어서...

하지만 여러분은 commit/push를 할 일이 거의 없을 겁니다. 대신 **pull**을 하게 되는데, 이 동작은 여러분이 컴퓨터로 clone한 repository 상태를 최신 버전으로 갱신하는 동작입니다. 한 번 따라해 보겠습니다.
> - Ctrl+Shift+P를 누르시거나, 상단 메뉴바에서 'Repository' > 'Pull'을 선택해봅시다.
> - 만약에 GitHub에서의 변경 사항이 존재한다면(=즉 제가 업데이트시켜 두었다면) 변경 사항을 반영할 것인지를 한 번 더 묻게 됩니다. 그러면 화면에 있는 'Pull Origin'을 누르시면 됩니다.
> - 끄읏.

# 2. 가상 환경 구축하기
가상 환경을 구축하는 이유는 python 버전 충돌을 피하기 위해서입니다. 저희가 사용할 웹 프레임워크(Framework)인 Django는 python 기반으로 돌아가는데, 이 때 서로 다른 버전의 Django를 사용하면 작업 결과물이 꼬일 수도 있습니다. 그래서 Django 3.1.3으로 버전을 통일하기 위해 가상 환경을 만든 다음, 그 가상 환경 안에서 Django 3.1.3을 설치해 서버를 구동시키게 됩니다.\
<br>순서는 아래와 같습니다. 어렵지 않으니까 천천히 따라와볼까요? :)

## 2.1. python 설치
python을 설치해야 Django도 설치할 수 있습니다. 구글 검색창에 python을 검색하면 공식 사이트가 나오는데, 여기에 들어가서 자신에게 맞는 python을 install합니다. 무엇을 설치해야 할 지 헷갈리신다면, 상단의 'Download'에 마우스를 올리자마자 나오는 것('Download for Windows, python 3.9.5')를 설치하셔도 됩니다. 물론 MacOS는 'Download'에서 'MacOS X'를 클릭하셔서 나오는 걸 설치하셔야 하지만요.\
python을 설치하실 때, **옵션 중에서 '환경 변수 추가하기'를 반드시 선택**해야 합니다. 만약에 까먹고 체크하지 않으셨다면, 다음을 따라해봅시다.

**가. 윈도우의 경우**
> - 제어판 > 프로그램 및 기능 > Python 3.9.X\
> 저 버전으로 Python이 설치되어 있을 겁니다. 참고로 Python Launcher가 아니니까 주의!
> - Python 3.X.X를 클릭한 다음, **'변경'** 클릭. 실수로 **삭제**를 클릭하시면 안 됩니다!
> - Modify 선택
> - Optional Features: **모두 다 체크 선택**하고 Next
> - Advanced Options: **'Precompile standard library' 빼고** 다 선택한 다음, Install
> - 끄읏!

**나. 맥os의 경우**
> - 맥은 환경변수보단 파이썬 버전을 체크해야 합니다. 맥은 기본적으로 파이썬 2를 제공하는데, 문제는 2는 지원이 중단되었고 우리는 3을 사용해야 한다는 점... 그래서 python 버전 3가 기본 파이썬인지 확인해야 합니다. 아래를 따라해봅시다.
> - bash를 엽니다. **bash**는 맥os에서 명령 프롬프트와 같은 기능을 합니다.
> - 다음을 입력해봅니다.
> ```bash
> python --version
> ```
> - 파이썬 3와 pip3가 있는지 확인해봅니다. 확인하셨으면 패쓰. 파이썬 2가 기본으로 설정되어 있다면, 아래를 마저 따라해봅니다. 일단, bash에서 다음을 입력해봅니다
> ```bash
> ls -l /usr/local/bin/python*
> ```
> - 뭔가 주루룩 뜰텐데, 아마 저를 따라하셨다면 python 3.9를 설치하셨을테니 다음을 입력해봅니다
> ```bash
> ls -s -f /usr/local/bin/python3.9 /usr/local/bin/python
> ```
> - 이제 다시 파이썬 버전을 체크해봅니다
> ```bash
> python --version
> ```
> 3.9.X로 되어 있으면 성공~!

## 3.2. 배치 파일(Batch File) 수정
우리의 작업 폴더 snumentalhealth_21_1을 보면, 확장자가 .cmd인 정체불명의 파일이 있습니다. 이 파일은 **배치 파일Batch File**이라고 부르는 친구로, 명령 프롬프트나 Bash에서 지정된 명령을 알아서 입력해주는 파일입니다. 즉, 일종의 단축키입니다.\
우리는 이 파일을 가상 환경 실행의 단축키로 사용할 예정입니다. 지금부터 'snumental_21_1.cmd'라는 파일을 다음과 같이 변경하겠습니다. 먼저, 이 파일을 잘라내서 바로 위에 있는 폴더에 붙여넣으세요. 그 다음에,

**가. 윈도우의 경우**
> 아래와 같이 수정합니다.
> ```cmd
> @echo off
> cd c:/venvs
> c:/venvs/mysite/Scripts/activate
> ```
> 그리고 저장합니다.

**나. 맥os의 경우**
> - 먼저 파일 확장자를 .sh로 변경합니다.
> - 그리고 이렇게 수정해봅시다.
> ```bash
> #!/bin/bash
> cd /venvs
> /venvs/mysite/bin/source .activate
> ```
> 그리고 저장합니다.
